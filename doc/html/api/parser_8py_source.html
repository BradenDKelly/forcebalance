00001 """ @package forcebalance.parser Input file parser for ForceBalance jobs.  Additionally, the location for all default options.
00002 
00003 Although I will do my best to write good documentation,
00004 for many programs the input parser becomes the most up-to-date
00005 source for documentation.  So this is a great place to write
00006 lots of comments for those who implement new functionality.
00007 
00008 There are two types of sections for options - GENERAL and TARGET.
00009 Since there can be many fitting targets within a single job (i.e. we
00010 may wish to fit water trimers and hexamers, which constitutes two
00011 fitting targets) the input is organized into sections, like so:
00012 
00013 $options\n
00014 gen_option_1 Big\n
00015 gen_option_2 Mao\n
00016 $target\n
00017 tgt_option_1 Sniffy\n
00018 tgt_option_2 Schmao\n
00019 $target\n
00020 tgt_option_1 Nifty\n
00021 tgt_option_2 Jiffy\n
00022 $end
00023 
00024 In this case, two sets of target options are generated in addition to the general option.
00025 
00026 (Note: "Target" used to be called "Simulation".  Backwards compatibility is maintained.)
00027 
00028 Each option is meant to be parsed as a certain variable type.
00029 
00030 - String option values are read in directly; note that only the first two words in the line are processed
00031 - Some strings are capitalized when they are read in; this is mainly for function tables like OptTab and TgtTab
00032 - List option types will pick up all of the words on the line and use them as values,
00033 plus if the option occurs more than once it will aggregate all of the values.
00034 - Integer and float option types are read in a pretty straightforward way
00035 - Boolean option types are always set to true, unless the second word is '0', 'no', or 'false' (not case sensitive)
00036 - Section option types are meant to treat more elaborate inputs, such
00037 as the user pasting in output parameters from a previous job as input,
00038 or a specification of internal coordinate system.  I imagine that for
00039 every section type I would have to write my own parser.  Maybe a
00040 ParsTab of parsing functions would work. :)
00041 
00042 To add a new option, simply add it to the dictionaries below and give it a default value if desired.
00043 If you add an entirely new type, make sure to implement the interpretation of that type in the parse_inputs function.
00044 
00045 @author Lee-Ping Wang
00046 @date 11/2012
00047 """
