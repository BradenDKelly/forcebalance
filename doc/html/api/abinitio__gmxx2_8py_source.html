27  """ForceBalance class for force and energy matching with the modified GROMACS.
28 
29  This class allows us to use a heavily modified version of GROMACS
30  (a major component of this program) to compute the objective
31  function contribution. The modified GROMACS does the looping
32  through snapshots, computes the interactions as well as the
33  derivatives, and sums them up to build the objective function.
34  I will write that documentation elsewhere, perhaps when I port
35  GROMACS over to version 4.5.4.
36 
37  This class implements the 'get_energy_force_' method. When it is
38  called, the force field is printed to the temporary directory
39  along with several files containing the information needed by the
40  modified GROMACS (the Boltzmann weights, the parameters that need
41  derivatives and their values, the QM energies and forces, and the
42  energy / force weighting.)
43 
44  The modified GROMACS is called with the arguments '-rerun all.gro
45  -fortune -rerunvsite' to loop over the snapshots, turn on force
46  matching functionality and reconstruct virtual site positions (an
47  important consideration if we're changing the virtual site
48  positions in the optimization). Its outputs are 'e2f2bc' which
49  means 'energy squared, force squared, boltzmann corrected' and
50  contains the objective function, 'a1dbc' and 'a2dbc' containing
51  analytic first and second derivatives, 'gmxboltz' containing the
52  Boltzmann weights used, and possibly some other stuff.
53 
54  Most importantly, 'e2f2bc', 'a1dbc' and 'a2dbc' are read by 'get'

168  def get_energy_force_(self,mvals,AGrad=False,AHess=False):
169  """ Calls the modified GROMACS and collects the objective function contribution.
170 
171  First we create the force field using the parameter values that were
172  passed in. Note that we may pass in physical parameters directly
173  and bypass the coordinate transformation by setting self.usepvals to True.
174 
175  The physical parameters are printed to 'pvals' for GROMACS to read -
176  of course GROMACS knows the parameters already, but this facilitates
177  retrieval from the low level subroutines.
178 
179  Several switches are printed to files, such as:
180  - 'FirstDerivativesOnly' to prevent computation of the Hessian
181  - 'NoDerivatives' to prevent computation of the Hessian AND the gradient
182 
183  GROMACS is called in the callgmxx2() method.
184 
185  The output files are then parsed for the objective function and its
186  derivatives are read in. The answer is passed out as a dictionary:
187  {'X': Objective Function, 'G': Gradient, 'H': Hessian}
188 
189  @param[in] mvals Mathematical parameter values
190  @param[in] AGrad Switch to turn on analytic gradient
191  @param[in] AHess Switch to turn on analytic Hessian
192  @return Answer Contribution to the objective function
193 
194  @todo Some of these files don't need to be printed, they can be passed
195  to GROMACS as arguments. Let's think about this some more.
196  @todo Currently I have no way to pass out the qualitative indicators.
197 
198  """
199  Answer = {}
200  cwd = os.getcwd()
201  # Create the new force field!!
202  pvals = self.FF.make(mvals,self.usepvals)
203  gmxx2_print("pvals",append([0],pvals),"double")
204  if AHess:
205  AGrad = True
206  remove_if_exists("FirstDerivativesOnly")
207  remove_if_exists("NoDerivatives")
208  elif AGrad:
209  with open("FirstDerivativesOnly",'w') as f: f.close()
210  remove_if_exists("NoDerivatives")
211  else:
212  with open("NoDerivatives",'w') as f: f.close()
213  print "\rGMXX2: %s\r" % self.name,
214  self.callgmxx2()
215  # Parse the output files
216  for line in open('e2f2bc').readlines():
217  sline = line.split()
218  if match('^Objective Function',line):
219  X = floatornan(sline[-1])
220  Answer['X'] = X
221  # Pass out the qualitative indicator
222  elif match('^Energy Error \(kJ/mol\)',line):
223  E = floatornan(sline[-1])
224  self.e_err = E
225  elif match('^Force Error',line):
226  F = floatornan(sline[-1])
227  self.f_err = F
228  # Derivatives in the physical parameter values
229  GR = array([float(i.split()[2]) for i in open('a1dbc').readlines()])
230  # If we're working in the rescaled space, rescale and transform to the 'mathematical' derivatives
231  # Otherwise just use the raw values.
232  if self.usepvals:
233  Answer['G'] = col(GR.copy())
234  else:
235  G = flat(mat(self.FF.tm)*col(GR.copy()))
236  Answer['G'] = G
237  # Reads in second derivatives from GROMACS
238  # Even if we're just calling the gradients, we want the diagonal Hessian elements as well!
239  HR = zeros((self.FF.np, self.FF.np))
240  for line in open('a2dbc').readlines():
241  i = int(line.split()[1])
242  j = int(line.split()[2])
243  HR[i, j] = float(line.split()[3])
244  if self.usepvals:
245  Answer['H'] = mat(HR.copy())
246  else:
247  H = mat(self.FF.tm)*mat(HR.copy())*mat(self.FF.tmI) # Projects charge constraints out of second derivative
248  Answer['H'] = H
249  return Answer
