00264 class Penalty:
00265     """ Penalty functions for regularizing the force field optimizer.
00266 
00267     The purpose for this module is to improve the behavior of our optimizer;
00268     essentially, our problem is fraught with 'linear dependencies', a.k.a.
00269     directions in the parameter space that the objective function does not
00270     respond to.  This would happen if a parameter is just plain useless, or
00271     if there are two or more parameters that describe the same thing.
00272 
00273     To accomplish these objectives, a penalty function is added to the
00274     objective function.  Generally, the more the parameters change (i.e.
00275     the greater the norm of the parameter vector), the greater the
00276     penalty.  Note that this is added on after all of the other
00277     contributions have been computed.  This only matters if the penalty
00278     'multiplies' the objective function: Obj + Obj*Penalty, but we also
00279     have the option of an additive penalty: Obj + Penalty.
00280 
00281     Statistically, this is called regularization.  If the penalty function
00282     is the norm squared of the parameter vector, it is called ridge regression.
00283     There is also the option of using simply the norm, and this is called lasso,
00284     but I think it presents problems for the optimizer that I need to work out.
00285 
00286     Note that the penalty functions can be considered as part of a 'maximum
00287     likelihood' framework in which we assume a PRIOR PROBABILITY of the
00288     force field parameters around their initial values.  The penalty function
00289     is related to the prior by an exponential.  Ridge regression corresponds
00290     to a Gaussian prior and lasso corresponds to an exponential prior.  There
00291     is also 'elastic net regression' which interpolates between Gaussian
00292     and exponential using a tuning parameter.
00293 
00294     Our priors are adjustable too - there is one parameter, which is the width
00295     of the distribution.  We can even use a noninformative prior for the
00296     distribution widths (hyperprior!).  These are all important things to
00297     consider later.
00298 
00299     Importantly, note that here there is no code that treats the distribution
00300     width.  That is because the distribution width is wrapped up in the
00301     rescaling factors, which is essentially a coordinate transformation
00302     on the parameter space.  More documentation on this will follow, perhaps
00303     in the 'rsmake' method.
00304 
00305     """

00308                       'FUSE' : 3, 'FUSION' : 3, 'FUSE_L0' : 4, 'FUSION_L0' : 4, 'FUSION-L0' : 4,

00338         X = Objective['X']
00339         G = Objective['G']
00340         H = Objective['H']
00341         np = len(mvals)
00342         K0, K1, K2 = self.Pen_Tab[self.ptyp](mvals)
