91 
92  The objective function is a combination of contributions from the different
93  fitting targets. Basically, it loops through the targets, gets their
94  contributions to the objective function and then sums all of them
95  (although more elaborate schemes are conceivable). The return value is the
96  same data type as calling the target itself: a dictionary containing
97  the objective function, the gradient and the Hessian.

100  too far from their initial values.

263 class Penalty:
264  """ Penalty functions for regularizing the force field optimizer.
265 
266  The purpose for this module is to improve the behavior of our optimizer;
267  essentially, our problem is fraught with 'linear dependencies', a.k.a.
268  directions in the parameter space that the objective function does not
269  respond to. This would happen if a parameter is just plain useless, or
270  if there are two or more parameters that describe the same thing.
271 
272  To accomplish these objectives, a penalty function is added to the
273  objective function. Generally, the more the parameters change (i.e.
274  the greater the norm of the parameter vector), the greater the
275  penalty. Note that this is added on after all of the other
276  contributions have been computed. This only matters if the penalty
277  'multiplies' the objective function: Obj + Obj*Penalty, but we also
278  have the option of an additive penalty: Obj + Penalty.
279 
280  Statistically, this is called regularization. If the penalty function
281  is the norm squared of the parameter vector, it is called ridge regression.
282  There is also the option of using simply the norm, and this is called lasso,
283  but I think it presents problems for the optimizer that I need to work out.
284 
285  Note that the penalty functions can be considered as part of a 'maximum
286  likelihood' framework in which we assume a PRIOR PROBABILITY of the
287  force field parameters around their initial values. The penalty function
288  is related to the prior by an exponential. Ridge regression corresponds
289  to a Gaussian prior and lasso corresponds to an exponential prior. There
290  is also 'elastic net regression' which interpolates between Gaussian
291  and exponential using a tuning parameter.
292 
293  Our priors are adjustable too - there is one parameter, which is the width
294  of the distribution. We can even use a noninformative prior for the
295  distribution widths (hyperprior!). These are all important things to
296  consider later.
297 
298  Importantly, note that here there is no code that treats the distribution
299  width. That is because the distribution width is wrapped up in the
300  rescaling factors, which is essentially a coordinate transformation
301  on the parameter space. More documentation on this will follow, perhaps
302  in the 'rsmake' method.
303 
304  """
305  Pen_Names = {'HYP' : 1, 'HYPER' : 1, 'HYPERBOLIC' : 1, 'L1' : 1, 'HYPERBOLA' : 1,
306  'PARA' : 2, 'PARABOLA' : 2, 'PARABOLIC' : 2, 'L2': 2, 'QUADRATIC' : 2,
307  'FUSE' : 3, 'FUSION' : 3, 'FUSE_L0' : 4, 'FUSION_L0' : 4, 'FUSION-L0' : 4,
308  'FUSE-BARRIER' : 5, 'FUSE-BARRIER' : 5, 'FUSE_BARRIER' : 5, 'FUSION_BARRIER' : 5}

336  def compute(self, mvals, Objective):
337  X = Objective['X']
338  G = Objective['G']
339  H = Objective['H']
340  np = len(mvals)
341  K0, K1, K2 = self.Pen_Tab[self.ptyp](mvals)
342  XAdd = 0.0
343  GAdd = zeros(np, dtype=float)
344  HAdd = zeros((np, np), dtype=float)
345  if self.fadd > 0.0:
346  XAdd += K0 * self.fadd
347  GAdd += K1 * self.fadd
348  HAdd += K2 * self.fadd
349  if self.fmul > 0.0:
350  XAdd += ( X*K0 ) * self.fmul
351  GAdd += array( G*K0 + X*K1 ) * self.fmul
352  GK1 = reshape(G, (1, -1))*reshape(K1, (-1, 1))
353  K1G = reshape(K1, (1, -1))*reshape(G, (-1, 1))
354  HAdd += array( H*K0+GK1+K1G+X*K2 ) * self.fmul
355  return XAdd, GAdd, HAdd

357  def L2_norm(self, mvals):
358  """
359  Harmonic L2-norm constraints. These are the ones that I use
360  the most often to regularize my optimization.
361 
362  @param[in] mvals The parameter vector
363  @return DC0 The norm squared of the vector
364  @return DC1 The gradient of DC0
365  @return DC2 The Hessian (just a constant)
366 
367  """
368  mvals = array(mvals)
369  DC0 = dot(mvals, mvals)
370  DC1 = 2*array(mvals)
371  DC2 = 2*eye(len(mvals))
372  return DC0, DC1, DC2

374  def HYP(self, mvals):
375  """
376  Hyperbolic constraints. Depending on the 'b' parameter, the smaller it is,
377  the closer we are to an L1-norm constraint. If we use these, we expect
378  a properly-behaving optimizer to make several of the parameters very nearly zero
379  (which would be cool).
380 
381  @param[in] mvals The parameter vector
382  @return DC0 The hyperbolic penalty
383  @return DC1 The gradient
384  @return DC2 The Hessian
385 
386  """
387  mvals = array(mvals)
388  np = len(mvals)
389  DC0 = sum((mvals**2 + self.b**2)**0.5 - self.b)
390  DC1 = mvals*(mvals**2 + self.b**2)**-0.5
391  DC2 = diag(self.b**2*(mvals**2 + self.b**2)**-1.5)
392  return DC0, DC1, DC2
