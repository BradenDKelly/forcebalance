00002 #  Penalty functions for regularizing the force field optimizer.
00003 # 
00004 # The purpose for this module is to improve the behavior of our optimizer;
00005 # essentially, our problem is fraught with 'linear dependencies', a.k.a.
00006 # directions in the parameter space that the objective function does not
00007 # respond to.  This would happen if a parameter is just plain useless, or
00008 # if there are two or more parameters that describe the same thing.
00009 # 
00010 # To accomplish these objectives, a penalty function is added to the
00011 # objective function.  Generally, the more the parameters change (i.e.
00012 # the greater the norm of the parameter vector), the greater the
00013 # penalty.  Note that this is added on after all of the other
00014 # contributions have been computed.  This only matters if the penalty
00015 # 'multiplies' the objective function: Obj + Obj*Penalty, but we also
00016 # have the option of an additive penalty: Obj + Penalty.
00017 # 
00018 # Statistically, this is called regularization.  If the penalty function
00019 # is the norm squared of the parameter vector, it is called ridge regression.
00020 # There is also the option of using simply the norm, and this is called lasso,
00021 # but I think it presents problems for the optimizer that I need to work out.
00022 # 
00023 # Note that the penalty functions can be considered as part of a 'maximum
00024 # likelihood' framework in which we assume a PRIOR PROBABILITY of the
00025 # force field parameters around their initial values.  The penalty function
00026 # is related to the prior by an exponential.  Ridge regression corresponds
00027 # to a Gaussian prior and lasso corresponds to an exponential prior.  There
00028 # is also 'elastic net regression' which interpolates between Gaussian
00029 # and exponential using a tuning parameter.
00030 # 
00031 # Our priors are adjustable too - there is one parameter, which is the width
00032 # of the distribution.  We can even use a noninformative prior for the
00033 # distribution widths (hyperprior!).  These are all important things to
00034 # consider later.
00035 # 
00036 # Importantly, note that here there is no code that treats the distribution
00037 # width.  That is because the distribution width is wrapped up in the
00038 # rescaling factors, which is essentially a coordinate transformation
00039 # on the parameter space.  More documentation on this will follow, perhaps
00040 # in the 'rsmake' method.
00041 # 
00042 # 
00043 
00044 from numpy import array, dot, eye, linalg, ones, reshape, zeros

00051 # 
00052 #     Harmonic L2-norm constraints.  These are the ones that I use
00053 #     the most often to regularize my optimization.
00054 # 
00055 #     @param[in] mvals The parameter vector
00056 #     @return DC0 The norm squared of the vector
00057 #     @return DC1 The gradient of DC0
00058 #     @return DC2 The Hessian (just a constant)
00059 # 
00060 #     
00061 def L2_norm(mvals):
00062     DC0 = dot(mvals, mvals)
00063     DC1 = 2*array(mvals)
00064     DC2 = 2*eye(len(mvals))
00065     return DC0, DC1, DC2
00066 
00067 ##
00068 # 
00069 #     Linear L1-norm constraints.  If we use these, we expect
00070 #     a properly-behaving optimizer to make several of the parameters zero
00071 #     (which would be cool).  However, I haven't tried it out yet.
00072 # 
00073 #     @param[in] mvals The parameter vector
00074 #     @return DC0 The norm of the vector
00075 #     @return DC1 The gradient of DC0, a constant
00076 #     @return DC2 The Hessian (zero)
00077 # 
00078 #     
00079 def L1_norm(mvals):
00080     np = len(mvals)
00081     DC0 = linalg.norm(mvals)
00082     DC1 = ones(len(mvals))
00083     DC2 = zeros((np, np), dtype=float)
00084     return DCO, DC1, DC2

00088            }

00091 # 
00092 #     This function is called by Project.Objective to actually impose the penalty
00093 #     on the objective function.  It allows us to switch between different penalty types
00094 #     (see Pen_Tab, I think we only have two options) and to choose either
00095 #     an additive penalty (adds to the objective function), a multiplicative penalty
00096 #     (multiplies the objective function by 1+P), or both.
00097 # 
00098 #     How to use: I can only imagine calling this from Project.Objective at the moment.
00099 # 
00100 #     Inputs:
00101 #     mvals        = The parameter vector
00102 #     Objective    = A dictionary of the objective function and its derivatives evaluated at the parameters
00103 #     Penalty_Type = A penalty functional form that comes out of Pen_Tab
00104 #     Factor_Add   = Prefactor for the strength of an additive penalty
00105 #     Factor_Mult  = Prefactor for the strength of a multiplicative penalty
00106 # 
00107 #     Outputs:
00108 #     XAdd         = The additional penalty
00109 #     GAdd         = The penalty's contribution to the gradient
00110 #     HAdd         = The penalty's contribution to the Hessian
00111 #     
00112 def Penalty(mvals, Objective, Penalty_Type, Factor_Add=0.0, Factor_Mult=0.0):
00113     X = Objective['X']
00114     G = Objective['G']
00115     H = Objective['H']
00116     np = len(mvals)
00117     K0, K1, K2 = Pen_Tab[Penalty_Type](mvals)
00118     XAdd = 0.0
00119     GAdd = zeros(np, dtype=float)
00120     HAdd = zeros((np, np), dtype=float)
00121     if Factor_Add > 0.0:
00122         XAdd += K0 * Factor_Add
00123         GAdd += K1 * Factor_Add
00124         HAdd += K2 * Factor_Add
00125     if Factor_Mult > 0.0:
00126         XAdd += ( X*K0 ) * Factor_Mult
00127         GAdd += array( G*K0 + X*K1 ) * Factor_Mult
00128         GK1 = reshape(G, (1, -1))*reshape(K1, (-1, 1))
00129         K1G = reshape(K1, (1, -1))*reshape(G, (-1, 1))
00130         HAdd += array( H*K0+GK1+K1G+X*K2 ) * Factor_Mult
00131     return XAdd, GAdd, HAdd
