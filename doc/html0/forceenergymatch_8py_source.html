00020 # 
00021 #     In force and energy matching, we introduce the following concepts:
00022 #     - The number of snapshots
00023 #     - The reference energies and forces (eqm, fqm) and the file they belong in (qdata.txt)

00110     ##
00111     #  Read the reference data (for force and energy matching)
00112     #         from a file such as qdata.txt.
00113     # 
00114     #         @todo Add an option for picking any slice out of
00115     #         qdata.txt, helpful for cross-validation
00116     #         
00117     #         @todo Closer integration of reference data with program -
00118     #         leave behind the qdata.txt format?  (For now, I like the
00119     #         readability of qdata.txt)
00120     # 
00121     #         After reading in the information from qdata.txt, it is converted
00122     #         into the GROMACS energy units (kind of an arbitrary choice);
00123     #         forces (kind of a misnomer in qdata.txt) are multipled by -1
00124     #         to convert gradients to forces.
00125     # 
00126     #         We also subtract out the mean energies of all energy arrays
00127     #         because energy/force matching does not account for zero-point
00128     #         energy differences between MM and QM (i.e. energy of electrons
00129     #         in core orbitals).
00130     # 
00131     #         The configurations in force/energy matching typically come
00132     #         from a the thermodynamic ensemble of the MM force field at
00133     #         some temperature (by running MD, for example), and for many
00134     #         reasons it is helpful to introduce non-Boltzmann weights in
00135     #         front of these configurations.  There are two options: WHAM
00136     #         Boltzmann weights (for combining the weights of several
00137     #         simulations together) and QM Boltzmann weights (for converting
00138     #         MM weights into QM weights).  Note that the two sets of weights
00139     #         'stack'; i.e. they can be used at the same time.
00140     # 
00141     #         A 'hybrid' ensemble is possible where we use 50% MM and 50% QM
00142     #         weights.  Please read more in LPW and Troy Van Voorhis, JCP
00143     #         Vol. 133, Pg. 231101 (2010), doi:10.1063/1.3519043.
00144     #         
00145     #         @todo The WHAM Boltzmann weights are generated by external
00146     #         scripts (wanalyze.py and make-wham-data.sh) and passed in;
00147     #         perhaps these scripts can be added to the ForceBalance
00148     #         distribution or integrated more tightly.
00149     # 
00150     #         Finally, note that using non-Boltzmann weights degrades the
00151     #         statistical information content of the snapshots.  This
00152     #         problem will generally become worse if the ensemble to which
00153     #         we're reweighting is dramatically different from the one we're
00154     #         sampling from.  We end up with a set of Boltzmann weights like
00155     #         [1e-9, 1e-9, 1.0, 1e-9, 1e-9 ... ] and this is essentially just
00156     #         one snapshot.  I believe Troy is working on something to cure
00157     #         this problem.
00158     # 
00159     #         Here, we have a measure for the information content of our snapshots,
00160     #         which comes easily from the definition of information entropy:
00161     # 
00162     #         S = -1*Sum_i(P_i*log(P_i))
00163     #         InfoContent = exp(-S)
00164     # 
00165     #         With uniform weights, InfoContent is equal to the number of snapshots;
00166     #         with horrible weights, InfoContent is closer to one.
00167     #

00247     # 
00248     #         LPW 01-11-2012
00249     #         
00250     #         This subroutine builds the objective function (and optionally
00251     #         its derivatives) from a general simulation software.  This is
00252     #         in contrast to using GROMACS-X2, which computes the objective
00253     #         function and prints it out; then 'get' only needs to call
00254     #         GROMACS and read it in.
00255     # 
00256     #         This subroutine interfaces with simulation software 'drivers'.
00257     #         The driver is only expected to give the energy and forces.
00258     # 
00259     #         Now this subroutine may sound trivial since the objective
00260     #         function is simply a least-squares quantity (M-Q)^2 - but
00261     #         there are a number of nontrivial considerations.  I will list
00262     #         them here.
00263     #         
00264     #         0) Polytensor formulation: Because there may exist covariance
00265     #         between different components of the force (or covariance
00266     #         between the energy and the force), we build the objective
00267     #         function by taking outer products of vectors that have the
00268     #         form [E F_1x F_1y F_1z F_2x F_2y ... ], and then we trace it
00269     #         with the inverse of the covariance matrix to get the objective
00270     #         function.
00271     # 
00272     #         1) Boltzmann weights and normalization: Each snapshot has its
00273     #         own Boltzmann weight, which may or may not be normalized.
00274     #         This subroutine does the normalization automatically.
00275     #         
00276     #         2) Subtracting out the mean energy gap: The zero-point energy
00277     #         difference between reference and fitting simulations is
00278     #         meaningless.  This subroutine subtracts it out.
00279     #         
00280     #         3) Hybrid ensembles: This program builds a combined objective
00281     #         function from both MM and QM ensembles, which is rigorously
00282     #         better than using a single ensemble.
00283     # 
00284     #         Note that this subroutine does not do EVERYTHING that
00285     #         GROMACS-X2 can do, which includes:
00286     #         
00287     #         1) Internal coordinate systems
00288     #         2) 'Sampling correction' (deprecated, since it doesn't seem to work)
00289     #         3) Analytic derivatives
00290     #         
00291     #         In the previous code (ForTune) this subroutine used analytic
00292     #         first derivatives of the energy and force to build the
00293     #         derivatives of the objective function.  Here I will take a
00294     #         simplified approach, because building the derivatives are
00295     #         cumbersome.  For now we will return the objective function
00296     #         ONLY.  A two-point central difference should give us the first
00297     #         and diagonal second derivative anyhow.
00298     # 
00299     #         @todo Parallelization over snapshots is not implemented yet
00300     # 
00301     #         @param[in] mvals Mathematical parameter values
00302     #         @param[in] AGrad Switch to turn on analytic gradient, useless here
00303     #         @param[in] AHess Switch to turn on analytic Hessian, useless here
00304     #         @param[in] tempdir Temporary directory for running computation
00305     #         @return Answer Contribution to the objective function
00306     #         
00307     def get(self, mvals, AGrad=False, AHess=False, tempdir=None):
00308         if tempdir == None:
00309             tempdir = self.tempdir
00310         abstempdir = os.path.join(self.root,self.tempdir)
00311         Answer = {}
00312         cwd = os.getcwd()
00313         # Create the new force field!!
00314         pvals = self.FF.make(tempdir,mvals,self.usepvals)
00315         # Go into the temporary directory
00316         os.chdir(os.path.join(self.root,tempdir))
00317 
00318         #======================================#
00319         #   Copied from the old ForTune code   #
00320         #======================================#
00321         NC   = 3*self.fitatoms
00322         NCP1 = 3*self.fitatoms+1
00323         #==============================================================#
00324         # Note: Because of hybrid ensembles, we form two separate      #
00325         # objective functions.  This means the code is (trivially)     #
00326         # doubled in length because there are two sets of Boltzmann    #
00327         # weights.  In principle I could write a general subroutine    #
00328         # for a single set of Boltzmann weights and call it twice, but #
00329         # that would require me to loop over the snapshots twice.      #
00330         # However, after the loop over snapshots, the subroutine that  #
00331         # builds the objective function (build_objective, above) is    #
00332         # called twice using the MM-ensemble and QM-ensemble           #
00333         # variables.                                                   #
00334         #                                                              #
00335         #            STEP 1: Form all of the arrays.                   #
00336         #                                                              #
00337         # Quantities pertaining to the covariance matrix.              #
00338         # An explanation of variable names follows.                    #
00339         # Z(Y)  = The partition function in the MM (QM) ensemble       #
00340         # M0_M  = The mean of the MM values in the MM ensemble         #
00341         # QQ_Q  = The unnormalized expected value of <Q(X)Q>           #
00342         # Later on we divide these quantities by Z to normalize,       #
00343         # and Q0(X)Q0 is subtracted from QQ to get the covariance.     #
00344         #==============================================================#
00345         Z       = 0
00346         Y       = 0
00347         Q = zeros(NCP1,dtype=float)
00348         M0_M    = zeros(NCP1,dtype=float)
00349         Q0_M    = zeros(NCP1,dtype=float)
00350         QQ_M    = zeros((NCP1,NCP1),dtype=float)
00351         M0_Q    = zeros(NCP1,dtype=float)
00352         Q0_Q    = zeros(NCP1,dtype=float)
00353         QQ_Q    = zeros((NCP1,NCP1),dtype=float)
00354         #==============================================================#
00355         # Objective function polytensors: This is formed in a loop     #
00356         # over snapshots by taking the outer product (Q-M)(X)(Q-M),    #
00357         # multiplying by the Boltzmann weight, and then summing.       #
00358         #==============================================================#
00359         SPiXi = zeros((NCP1,NCP1),dtype=float)
00360         SRiXi = zeros((NCP1,NCP1),dtype=float)
00361         QBN = dot(self.qmboltz_wts[:self.ns],self.whamboltz_wts[:self.ns])
00362         #==============================================================#
00363         #             STEP 2: Loop through the snapshots.              #
00364         #==============================================================#
00365         for i in range(self.ns):
00366             print "Shot %i\r" % i,
00367             # Build Boltzmann weights and increment partition function.
00368             P   = self.whamboltz_wts[i]
00369             Z  += P
00370             R   = self.qmboltz_wts[i]*self.whamboltz_wts[i] / QBN
00371             Y  += R
00372             # Recall reference (QM) data
00373             Q[0] = self.eqm[i]
00374             Q[1:] = self.fqm[i,:].copy()
00375             # Increment the average quantities.
00376             QQ     = outer(Q,Q)
00377             # Call the simulation software to get the MM quantities.
00378             M = self.energy_force_driver(i)
00379             # Increment the average values.
00380             M0_M += P*M
00381             Q0_M += P*Q
00382             QQ_M += P*QQ
00383             M0_Q += R*M
00384             Q0_Q += R*Q
00385             QQ_Q += R*QQ
00386             # Increment the objective function.
00387             Xi  = outer(M,M) - 2*outer(Q,M) + outer(Q,Q)
00388             SPiXi += P * Xi
00389             SRiXi += R * Xi
00390         #==============================================================#
00391         #     STEP 3: Build the covariance matrix and invert it.       #
00392         #==============================================================#
00393         # The "two" is of no importance, it has been kept historically.
00394         EFW     = 2 *self.efweight
00395         CEFW    = 2 - EFW
00396         # Build the weight matrix, so the force contribution is suppressed by 1/3N
00397         WM      = zeros((NCP1,NCP1),dtype=float)
00398         WM[0,0] = sqrt(EFW)
00399         for i in range(1,NCP1):
00400             WM[i,i] = sqrt(EFW/NC)
00401         #==============================================================#
00402         #                      The covariance matrix                   #
00403         #                                                              #
00404         # Note: There is a detail here that might appear peculiar :    #
00405         # I'm building only one covariance matrix from both ensembles. #
00406         # This is the proper thing to do, believe me.                  #
00407         # The functional form looks like (X2_M + X2_Q)(C)^-1           #
00408         # I will build X2_M C^-1 and X2_Q C^-1 separately.             #
00409         #==============================================================#
00410         QBP  = self.qmboltz
00411         MBP  = 1 - self.qmboltz
00412         C    = MBP*(QQ_M-outer(Q0_M,Q0_M)/Z)/Z + QBP*(QQ_Q-outer(Q0_Q,Q0_Q)/Y)/Y
00413         if self.covariance == False:
00414             C = diag(C)
00415             C[1:] = mean(C[1:])
00416             C = diag(C)
00417         Ci   = invert_svd(C)
00418         # Get rid of energy-force covariance
00419         for i in range(1,NCP1):
00420             Ci[0,i] = 0.0;
00421             Ci[i,0] = 0.0;
00422         WCiW = array(mat(WM) * mat(Ci) * mat(WM)) # Weighted covariance matrix.
00423         #==============================================================#
00424         # STEP 4: Build the objective function and its derivatives by  #
00425         # tracing with the covariance matrix.  (Note that without      #
00426         # Sampling Correction, covariance matrix has no derivatives.   #
00427         # I am thankful for this.)  I will build X2_M C^-1 and X2_Q    #
00428         # C^-1 separately because it's cleaner that way.               #
00429         #==============================================================#
00430         X2_M  = build_objective(SPiXi,WCiW,Z,Q0_M,M0_M,NCP1)
00431         X2_Q  = build_objective(SRiXi,WCiW,Y,Q0_Q,M0_Q,NCP1)
00432         #==============================================================#
00433         #            STEP 5: Build the return quantities.              #
00434         #==============================================================#
00435         # The un-penalized objective function
00436         BC   = MBP * X2_M    + QBP * X2_Q
