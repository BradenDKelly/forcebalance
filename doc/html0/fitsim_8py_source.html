00012 # 
00013 #     Base class for all fitting simulations.
00014 #     
00015 #     In ForceBalance a 'fitting simulation' is defined as a simulation
00016 #     which computes a quantity that we can compare to a reference.  The
00017 #     force field parameters are tuned to reproduce the reference value as
00018 #     closely as possible.
00019 #     
00020 #     The 'computable quantities' may include energies and forces where the
00021 #     reference values come from QM calculations (energy and force matching),
00022 #     energies from an EDA analysis (Maybe in the future, FDA?), molecular
00023 #     properties (like polarizability, refractive indices, multipole moments
00024 #     or vibrational frequencies), relative entropies, and bulk properties.
00025 #     Single-molecule or bulk properties can even come from the experiment!
00026 #     
00027 #     The central idea in ForceBalance is that each quantity makes a
00028 #     contribution to the overall objective function.  So we can build force
00029 #     fields that fit several quantities at once, rather than putting all of
00030 #     our chips behind energy and force matching.  In the future
00031 #     ForceBalance may even include multiobjective optimization into the
00032 #     optimizer.
00033 #     
00034 #     The optimization is done by way of minimizing an 'objective
00035 #     function', which is comprised of squared differences between the
00036 #     computed and reference values.  These differences are not computed
00037 #     in this file, but rather in subclasses that use FittingSimulation
00038 #     as a base class.  Thus, the contents of FittingSimulation itself
00039 #     are meant to be as general as possible, because the pertinent
00040 #     variables apply to all types of fitting simulations.
00041 # 
00042 #     An important node: FittingSimulation requires that all subclasses
00043 #     have a method get(self,mvals,AGrad=False,AHess=False,tempdir=None)
00044 #     that does the following:
00045 #     
00046 #     Inputs: 
00047 #     mvals        = The parameter vector, which modifies the force field
00048 #     (Note to self: We include mvals with each FitSim because we can create
00049 #     copies of the force field and do finite difference derivatives)
00050 #     AGrad, AHess = Boolean switches for computing analytic gradients and Hessians
00051 #     tempdir      = Temporary directory; we can create multiple of these
00052 #     for parallelization of our jobs (a future consideration)
00053 # 
00054 #     Outputs:
00055 #     Answer       = {'X': Number, 'G': numpy.array(np), 'H': numpy.array((np,np)) }
00056 #     'X'          = The objective function itself
00057 #     'G'          = The gradient, elements not computed analytically are zero
00058 #     'H'          = The Hessian, elements not computed analytically are zero
00059 # 
00060 #     This is the only global requirement of a FittingSimulation.
00061 #     Obviously 'get' itself is not defined here, because its
00062 #     calculation will depend entirely on specifically which simulation
00063 #     we wish to run.  However, this should give us a unified framework
00064 #     which will faciliate rapid implementation of FittingSimulations.
00065 # 
00066 #     Future work:
00067 #     Robert suggested that I could enable automatic detection of which
00068 #     parameters need to be computed by finite difference.  Not a bad idea. :)
00069 # 
00070 #

00151     ##
00152     # Computes the objective function contribution and its gradient.
00153     # 
00154     #         First the low-level 'get' method is called with the analytic gradient
00155     #         switch turned on.  Then we loop through the fd1_pids and compute
00156     #         the corresponding elements of the gradient by finite difference,
00157     #         if the 'fdgrad' switch is turned on.  Alternately we can compute
00158     #         the gradient elements and diagonal Hessian elements at the same time
00159     #         using central difference if 'fdhessdiag' is turned on.
00160     #         
00161     def get_G(self,mvals=None):
00162         Ans = self.get(mvals,1,0)
00163         for i in range(self.FF.np):
00164             if any([j in self.FF.plist[i] for j in self.fd1_pids]) or 'ALL' in self.fd1_pids:
00165                 if self.fdgrad:
00166                     Ans['G'][i] = f1d2p(fdwrap_G(self,mvals,i),self.h,f0 = Ans['X'])
00167                 elif self.fdhessdiag:
00168                     Ans['G'][i], Ans['H'][i,i] = f12d3p(fdwrap_G(self,mvals,i),self.h,f0 = Ans['X'])
00169         # Additional call to build qualitative indicators
00170         self.get(mvals,0,0)
00171         self.gct += 1
00172         return Ans

00174     ##
00175     # Computes the objective function contribution and its gradient / Hessian.
00176     # 
00177     #         First the low-level 'get' method is called with the analytic gradient
00178     #         and Hessian both turned on.  Then we loop through the fd1_pids and compute
00179     #         the corresponding elements of the gradient by finite difference,
00180     #         if the 'fdgrad' switch is turned on.
00181     # 
00182     #         This is followed by looping through the fd2_pids and computing the corresponding
00183     #         Hessian elements by finite difference.  Forward finite difference is used
00184     #         throughout for the sake of speed.
00185     #         
00186     def get_H(self,mvals=None):
00187         Ans = self.get(mvals,1,1)
00188         if self.fdhess:
00189             for i in range(self.FF.np):
00190                 if any([j in self.FF.plist[i] for j in self.fd1_pids]) or 'ALL' in self.fd1_pids:
00191                     Ans['G'][i] = f1d2p(fdwrap_G(self,mvals,i),self.h,f0 = Ans['X'])
00192             for i in range(self.FF.np):
00193                 if any([j in self.FF.plist[i] for j in self.fd2_pids]) or 'ALL' in self.fd2_pids:
00194                     FDSlice = f1d2p(fdwrap_H(self,mvals,i),self.h,f0 = Ans['G'])
00195                     Ans['H'][i,:] = FDSlice
00196                     Ans['H'][:,i] = FDSlice
00197         elif self.fdhessdiag:
00198             for i in range(self.FF.np):
00199                 Ans['G'][i], Ans['H'][i,i] = f12d3p(fdwrap_G(self,mvals,i),self.h)
00200         # This builds the qualitative indicators
00201         self.get(mvals,0,0)
00202         self.hct += 1
00203         return Ans
